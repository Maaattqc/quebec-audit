# CHECKLIST PRÉ-PITCH — Vérification avant de montrer la démo au client

## Utilisation
Passe cette checklist AVANT chaque rencontre client.
Si un item fail, corrige-le avant le pitch. Un seul bug visible = crédibilité perdue.

---

## 1. CONTENU — Rien de faux

- [ ] Tous les numéros de téléphone sont corrects (vérifier vs le site actuel)
- [ ] L'adresse est exacte
- [ ] Le numéro RBQ/licence est le bon (ou absent si ils n'en ont pas)
- [ ] Les noms d'employés/propriétaires sont corrects (ou absents)
- [ ] Les services listés existent vraiment sur leur site
- [ ] Les heures d'ouverture sont correctes
- [ ] L'email est le bon
- [ ] Aucune statistique inventée (années, projets, etc.)
- [ ] Les témoignages sont identifiés comme exemples ("à remplacer")
- [ ] Les liens réseaux sociaux pointent vers leurs vrais comptes

## 2. RESPONSIVE — Tester sur vrais appareils

- [ ] **Mobile 375px** : ouvrir sur ton téléphone, pas juste les DevTools
- [ ] Le menu hamburger s'ouvre et se ferme
- [ ] Le hero est lisible (texte pas coupé, CTA visible)
- [ ] Les stats ne débordent pas
- [ ] Le formulaire est utilisable au pouce
- [ ] Aucun scroll horizontal (le test #1 que les gens remarquent)
- [ ] **Tablette** : tester en mode portrait ET paysage
- [ ] **Desktop** : tester à 1920px, rien d'étiré
- [ ] Les images ne sont pas déformées à aucune taille

## 3. FONCTIONNEL — Tout clique

- [ ] Le bouton téléphone ouvre bien l'app d'appel (tester sur mobile)
- [ ] Le lien email ouvre le client mail
- [ ] Les ancres (#services, #contact) scrollent au bon endroit
- [ ] Le formulaire affiche "Merci" quand on clique Envoyer
- [ ] Si Formspree est branché : envoyer un vrai test et vérifier ta boîte mail
- [ ] La nav change bien au scroll (transparent → solide)
- [ ] Les animations se déclenchent au scroll (pas toutes en même temps au load)
- [ ] Les compteurs s'animent quand ils apparaissent à l'écran
- [ ] Le select du type de projet fonctionne et affiche les options

## 4. VISUEL — Rien de cassé

- [ ] Aucun texte gris pâle illisible sur fond clair
- [ ] Les fonts Google sont bien chargées (pas de fallback Arial visible)
- [ ] Leur vidéo/image de fond se charge correctement
- [ ] Leur image dans la section "À propos" se charge
- [ ] Pas d'images cassées (icône broken image)
- [ ] Les hover states fonctionnent (services, cartes témoignages, boutons)
- [ ] Le marquee défile sans saccade
- [ ] Aucun élément qui overlap un autre
- [ ] Le site charge en moins de 3 secondes

## 5. AVANT LE MEETING — Préparation

- [ ] Ouvrir le site ACTUEL du client dans un onglet (pour comparer en live)
- [ ] Ouvrir ta démo dans un autre onglet
- [ ] Préparer la liste des changements (ce qu'on a gardé vs amélioré)
- [ ] Avoir le prix prêt (Starter vs Premium si applicable)
- [ ] Tester la connexion internet du lieu de meeting
- [ ] Si tu présentes sur un projecteur/TV : tester la résolution avant
- [ ] Avoir le message de suivi prêt (email ou SMS post-meeting)

## 6. APRÈS LE PITCH — Suivi

- [ ] Envoyer un email/SMS de remerciement dans les 2h
- [ ] Inclure un lien vers la démo si hébergée (ou screenshot)
- [ ] Mettre à jour le statut dans le hub de prospection
- [ ] Planifier une relance si pas de réponse (3-5 jours)

---

## Quick-test en 2 minutes

Si t'as pas le temps de tout vérifier, fais AU MINIMUM :

1. Ouvre sur ton téléphone → scroll complet → aucun bug visible?
2. Clique le bouton téléphone → ça appelle?
3. Soumets le formulaire → ça dit merci?
4. Compare côte à côte avec leur vrai site → la démo est clairement meilleure?

Si ces 4 passent, tu peux pitcher.
