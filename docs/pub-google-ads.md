# Strategie publicitaire — Google Ads

## Regle #1 : Ne PAS mettre de pub tout de suite

Le SEO est le canal principal. La pub sert a **valider la demande** et **accelerer** les premiers clients, pas a remplacer le SEO.

| Phase | Budget/mois | Objectif |
|---|---|---|
| Mois 1 | 0$ | Build + launch (zero pub) |
| Mois 2-3 | 50-100$ | Tester 3-5 outils, mesurer conversions |
| Mois 4-6 | 100-300$ | Scale les outils rentables |
| Mois 6+ | Reinvestir les profits | Le SEO devrait couvrir la majorite du trafic |

---

## Calcul de rentabilite

```
Cout par clic Google Ads : ~1-3$ (keywords Quebec = pas cher vs USA)
Taux de conversion : ~5-10% (visiteur → achat rapport)
Prix rapport : 15-25$

Exemple : 100 clics x 2$/clic = 200$ depenses
         100 clics x 7% conversion = 7 rapports
         7 rapports x 20$ = 140$ revenus

→ Pas rentable a 2$/clic et 7% conversion pour un rapport a 20$
→ MAIS rentable si :
  - Rapport a 25$+
  - OU conversion a 10%+
  - OU cout/clic a 1$
  - ET les clients reviennent (abo Pro) = valeur long-terme (LTV)
```

## Pub vs SEO : comparaison

| | Pub (Google Ads) | SEO (organique) |
|---|---|---|
| Quand ca marche | Immediatement | 3-6 mois |
| Cout | Chaque clic coute | Gratuit apres l'effort initial |
| Scalabilite | Lineaire (plus tu paies, plus tu as) | Exponentiel (compound) |
| Quand arreter | Tu paies = trafic. Tu arretes = zero. | Le trafic continue meme si tu fais rien |
| Role | **Valider la demande** + premiers clients | **Revenue principal** long-terme |

### Landing pages

Chaque campagne mene vers la page de l'outil (pas vers la homepage) :
- `/verifier-entrepreneur` pour ContractorCheck
- `/zone-inondable` pour FloodCheck
- etc.

La landing page doit :
1. Montrer le resultat gratuit immediatement (input → resultat basique)
2. Upsell vers le rapport payant
3. Avoir un CTA clair ("Obtenir le rapport complet — 19.99$")
4. Temoignages / nombre de rapports generes (social proof)

## Retargeting

### Quand l'activer

Pas au lancement. Activer quand :
- Le site a 500+ visiteurs/mois
- Le taux de conversion est mesure (on sait combien de gens achetent)

### Comment

1. **Google Ads Remarketing** : quelqu'un visite `/verifier-entrepreneur` sans acheter → pub "Votre rapport est pret" pendant 7 jours
2. **Facebook/Meta** : meme principe, retarget les visiteurs du site
3. **Budget retargeting** : 20-30% du budget total (CPC moins cher car audience chaude)

## Metriques a suivre

| Metrique | Cible |
|---|---|
| CPC (cout par clic) | < 2$ |
| CTR (taux de clic sur l'annonce) | > 3% |
| Taux de conversion (clic → achat) | > 5% |
| CPA (cout par acquisition) | < prix du rapport |
| ROAS (return on ad spend) | > 2x |

### Quand couper une campagne

- CPA > prix du rapport pendant 2 semaines → couper
- CTR < 1% → retravailler les annonces
- Taux de conversion < 2% → retravailler la landing page

## Framework A/B testing des annonces

### Principes

1. **2-3 variantes par ad group** : jamais plus, sinon le budget se dilue
2. **1 variable a la fois** : tester headline OU description OU CTA, pas tout en meme temps
3. **Minimum 100 conversions** avant de declarer un gagnant (ou 1,000 clics si le taux de conversion est faible)
4. **Review hebdo** : chaque lundi, verifier les performances et pauser les perdants

### Process de test

```
Semaine 1-2 : Lancer 2-3 variantes (headline differents)
Semaine 3   : Review → pauser la pire variante
Semaine 4   : Si pas de gagnant clair → continuer 1 semaine
Semaine 5   : Declarer gagnant → creer 2 nouvelles variantes basees sur le gagnant
              (tester description cette fois)
```

### Quoi tester en priorite

| Priorite | Element | Impact attendu |
|---|---|---|
| 1 | Headlines (titre principal) | +20-40% CTR |
| 2 | CTA (appel a l'action) | +10-20% conversion |
| 3 | Prix dans l'annonce vs pas de prix | +/- 15% conversion |
| 4 | Extensions (sitelinks, callouts) | +5-15% CTR |
| 5 | Landing page (au-dessus du pli) | +10-30% conversion |

## Smart Bidding vs encheres manuelles

### Decision table

| Mois | Strategie | Condition | Pourquoi |
|---|---|---|---|
| 1-2 | **Manuel (CPC max)** | < 30 conversions | Pas assez de data pour l'algorithme |
| 3-4 | **Maximize Conversions** | 30+ conversions totales | L'algo commence a apprendre |
| 4-6 | **Target CPA** | 50+ conversions/mois | Definir un CPA cible base sur les donnees |
| 6+ | **Target ROAS** | Pour les outils chers (PropTech, Pro) | Optimiser le retour, pas juste le volume |

### CPC max recommande au lancement (manuel)

| Campagne | CPC max | Justification |
|---|---|---|
| ContractorCheck | 2.00$ | Intent fort, bon taux de conversion |
| FloodCheck | 1.50$ | CPC naturellement bas |
| GarderieFind | 1.00$ | Volume eleve, conversion moyenne |
| PropTech Report | 3.00$ | Rapport cher (25$), marge elevee |
| TerraCheck | 1.50$ | CPC bas, volume moyen |

### Transition vers Target CPA

```
Etape 1 : Accumuler 30+ conversions en CPC manuel (2-4 semaines)
Etape 2 : Calculer le CPA moyen des 30 dernieres conversions
Etape 3 : Definir Target CPA = CPA moyen actuel (pas plus agressif)
Etape 4 : Laisser tourner 2 semaines sans toucher
Etape 5 : Si CPA respecte → baisser de 10%. Si CPA depasse → remonter de 10%.
```

## Suivi des conversions (GA4 + Stripe)

### Events GA4 a configurer

| Event | Trigger | Parametres |
|---|---|---|
| `view_tool` | Page outil chargee | `tool_name`, `city` |
| `use_tool_free` | Resultat gratuit affiche | `tool_name`, `query` |
| `click_buy` | Clic sur "Rapport complet" | `tool_name`, `price` |
| `begin_checkout` | Stripe Checkout ouvert | `tool_name`, `price`, `currency: CAD` |
| `purchase_complete` | Webhook Stripe `checkout.session.completed` | `tool_name`, `price`, `transaction_id` |
| `subscribe` | Webhook Stripe `customer.subscription.created` | `plan_name`, `price`, `interval` |

### Webhook Stripe → GA4 Measurement Protocol

```typescript
// Dans le webhook Stripe (server.ts)
app.post('/api/stripe/webhook', async (req, res) => {
  const event = stripe.webhooks.constructEvent(req.body, sig, endpointSecret);

  if (event.type === 'checkout.session.completed') {
    const session = event.data.object;

    // Envoyer l'event a GA4 via Measurement Protocol
    await fetch('https://www.google-analytics.com/mp/collect?' +
      `measurement_id=${GA4_ID}&api_secret=${GA4_SECRET}`, {
      method: 'POST',
      body: JSON.stringify({
        client_id: session.metadata.ga_client_id,
        events: [{
          name: 'purchase',
          params: {
            transaction_id: session.id,
            value: session.amount_total / 100,
            currency: 'CAD',
            items: [{ item_name: session.metadata.tool_name }],
          },
        }],
      }),
    });
  }
});
```

### Import conversions dans Google Ads

```
Etape 1 : GA4 > Admin > Data Streams > Web > Measurement Protocol API secret (creer)
Etape 2 : GA4 > Admin > Product Links > Google Ads (lier les comptes)
Etape 3 : Google Ads > Tools > Conversions > Import > Google Analytics 4
Etape 4 : Selectionner events : purchase_complete, subscribe
Etape 5 : Attribution model : Data-driven (ou Last click si < 300 conversions/mois)
Etape 6 : Conversion window : 30 jours (click-through), 1 jour (view-through)
Etape 7 : Valeur : utiliser la valeur dynamique du event (prix reel)
Etape 8 : Verifier dans Google Ads > Tools > Conversions que les conversions remontent
```

## Performance Max

### Quand utiliser Performance Max

- **Pas au lancement** — seulement apres mois 4+ quand on a des donnees de conversion solides
- **B2C visuels** : GarderieFind, LouerSmart, SalaireLab (les produits ou des visuels/videos aident)
- **Pas pour B2B** : les outils B2B convertissent mieux avec du Search classique (intent > visibilite)

### Asset groups

Chaque outil en Performance Max a son propre asset group :

```
Asset Group: GarderieFind
  - Textes : 5 headlines + 5 long headlines + 5 descriptions
  - Images : 3 paysage (1200x628), 3 carre (1200x1200), 1 portrait (960x1200)
  - Videos : 1 video YouTube 15-30 sec (optionnel mais recommande)
  - Logo : logo du site
  - URL finale : /garderie
  - Audience signal : parents 25-45 ans, interets garderie/famille, Quebec
```

### Budget Performance Max

| Mois | Budget PMax | % du budget total |
|---|---|---|
| 4-5 | 20$ | 10% |
| 6-8 | 50-100$ | 20% |
| 9+ | 100-200$ | 25-30% |

### Regles

- **Exclure branded search** : les recherches de marque doivent rester dans une campagne Search dediee (CPC moins cher, meilleur controle)
- **Audience signals** : configurer des signaux (pas du ciblage strict) — Google les utilise comme point de depart
- **Minimum 4 semaines** avant d'evaluer les resultats (l'algo apprend)
- **Ne pas mixer B2B et B2C** dans le meme asset group

### Implementation

```
Budget mensuel de base : 300$

Peak season (mars-juin, aout-sept) :
  300$ x 1.35 = 405$/mois (budget shift +35% moyen)

Low season (juillet, decembre) :
  300$ x 0.80 = 240$/mois

Budget annuel total : ~3,800$
```

### Regles de budget shift

- **Maximum +40%** sur un mois (pas de spikes trop violents, l'algo Google n'aime pas)
- **Minimum -20%** (garder un baseline pour continuer a collecter des donnees)
- **Rebalancer chaque trimestre** selon les performances reelles vs projections
- **Peak season** : augmenter budget + bids sur les campagnes saisonnieres ET baisser les autres
- Ne pas couper completement une campagne pendant le low season — juste reduire
